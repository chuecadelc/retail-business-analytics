# Retail Business Analytics Case Study
## Matrix Manufacturing & Retail (MMR) — SQL & Python Analysis

This project presents a business analytics case study for a fictional bicycle retail company, Matrix Manufacturing & Retail (MMR), developed as the final project for a Level 4 Data Analytics Skills Bootcamp (Distinction, Gateshead College, March 2026).

The original brief asked for a general sales report. I extended the scope significantly as I wanted to go step further and maximise this learning opportunity. This involved designing a realistic synthetic dataset from scratch, implementing a full data strategy and pipeline and delivering a 7-page business analytics report covering product performance, bike sales trends and customer loyalty analysis across six UK locations.

## Business Questions Addressed

- How are product sales performing across 10 product categories over 2022?
- How are bike sales (mountain, road and touring) trending across the year?
- How is the customer loyalty programme performing across UK locations?
- What strategic recommendations can be made to improve revenue and reduce customer churn?

## Project Highlights

- Designed a synthetic relational database from scratch in MySQL, including custom columns and variables not present in the original dataset
- Implemented a complete data pipeline: database creation → SQL extraction → CSV export → Python analysis → business report
- Applied four types of business analytics: descriptive, diagnostic, predictive and prescriptive
- Produced four visualisations in Python (Seaborn, Matplotlib) including line plots, a heatmap, and an interactive treemap
- Framed analysis around real business KPIs including profit margins, customer retention and market share by location

## Data Strategy & Pipeline

The project follows a structured data strategy covering:

- **Data governance** — GDPR compliance, data ownership, and documentation standards
- **Data pipeline** — raw data ingestion → SQL transformation → CSV export → Python analysis
- **Stakeholder mapping** — data scientist, data analyst and senior executive reporting chain

Three synthetic SQL tables were designed and populated:

| Table | Key Fields |
|-------|-----------|
| Customers | Customer ID, name, location, purchases, loyalty status, join date |
| Employees | Employee ID, name, working pattern, location |
| Products | Product ID, name, type, inventory, costs, sales volume, retail price |

## Key Findings

- Product sales across 2022 were largely seasonal with no consistent upward trend, suggesting gaps in marketing strategy across the year
- Mountain bike sales declined sharply after a strong start to the year, while road bike sales showed clear seasonal peaks in spring, summer and autumn
- The loyalty programme shows healthy sign-up rates across all six locations; Manchester and Newcastle had the largest customer bases while Birmingham and Glasgow had the highest average purchases among Gold tier members — suggesting a smaller but more loyal customer base in those locations
- Strategic recommendation: replicate the engagement approaches from high-average-purchase locations (Birmingham, Glasgow) to higher-volume locations (Manchester, Newcastle) to improve Gold tier retention

## How to Run

### Prerequisites

- MySQL Workbench 8.0 or above
- Python 3.x with the following packages:

```bash
pip install pandas numpy seaborn matplotlib squarify
```

### Steps

1. Run `01_database_creation.sql` in MySQL Workbench to create and populate the synthetic database
2. Run SQL scripts `02` and `03` to extract and export the relevant tables as CSV files
3. Run `04_analysis_and_visualisation.py` to reproduce all figures from the report
